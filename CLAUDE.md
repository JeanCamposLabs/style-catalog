# CLAUDE.md — working on this repo

Orientation for an AI agent **developing** the Style Catalog. If your job is to
*consume* the catalog (use an effect in another project), read
[`AGENTS.md`](AGENTS.md) instead — that's the consumer contract.

## What this is

A museum + machine-readable catalogue of self-contained web-design effects.
Each effect is one runnable HTML file with an embedded metadata block; a
zero-dependency static gallery (`index.html` + `assets/`) browses them, and a
build step generates the public data files. It deploys to GitHub Pages on every
push to `main`.

**The library is the source of truth. Generated files are never hand-edited.**

## Build pipeline

`npm run build` (`scripts/build.mjs`) scans `effects/`, validates every file,
and emits `catalog.json`, `assets/catalog.js`, the `api/` tree, and the deploy
stamp. If any effect fails validation the build aborts — so a green build means
a valid library. `npm run validate` runs the same checks without writing.

CI (`.github/workflows/ci.yml`) runs two install-free jobs: **quality**
(`npm run lint` = zero-dep syntax gate, `npm test` = Node's built-in runner over
`test/*.test.mjs`) and **validate** (`validate` + `build`, failing if the
generated files aren't committed and up to date). Always `npm run build` and
commit the result alongside any change to effects or `scripts/lib/catalog.mjs`.

## Gallery architecture (`assets/app.js`)

One IIFE, plain ES5-style vanilla JS (no framework, no build step, `var` +
function declarations — match that style). Section banners (`/* ---------- … */`)
mark the parts:

- **helpers / stats / filters sidebar / quick chips** — chrome + faceted filtering
- **matching** — search + filter predicate, featured/wow sort
- **grid** — `card()` builds each card; cards rest on a poster `<img>` and mount
  the live iframe **only on hover/focus** (`mountFrame`, `is-live` class). Missing
  posters fall back to the frozen iframe (`no-poster`).
- **modal** — tabs: Preview / Tune / Source / AI Notes / Details. `detailsHtml`
  renders metadata incl. clickable **Related** chips; **Copy for AI**
  (`buildEffectMarkdown`) and the bundle exporter (`buildBundleMarkdown`) produce
  paste-ready prompts.
- **color helpers / palette bar** — live re-skinning via CSS custom properties
  (`--accent`, `--accent-2`, …); `applyBrandEverywhere` repaints everything.
- **hash routing** — deep links (`#effect=<id>`, filters) via `setHash`/`applyHash`.
- **bundle (cart)** — multi-select → one combined markdown bundle.
- **spaceship cursor** (`initShipCursor`) — canvas exhaust trail + an SVG ship
  that eases toward the pointer (time-based easing) and banks into motion.
  **No CSS `filter`** anywhere here on purpose: a drop-shadow on the
  rotating/rescaling ship re-rasterizes every frame and tanks the frame rate, so
  the glow is baked into the SVG as a radial gradient. Gated to fine pointers +
  motion (touch / reduced-motion keep the native cursor).

## Dev workflow

```bash
npm run new -- <theme-slug> <effect-slug> "Title"  # scaffold an effect
# edit effects/<theme-slug>/<effect-slug>.html (metadata + style + markup)
npm run build && npm run validate                   # regenerate + check
npm run serve                                        # preview → localhost:4321
```

Regenerate poster thumbnails after adding/changing effects (Playwright is a
**dev-only** dependency; the runtime gallery stays zero-dep, and CI never
installs it):

```bash
npm i -D playwright && npx playwright install chromium
npm run posters    # writes assets/posters/<id>.jpg — commit them
```

## Release / deploy ritual

Pushing to `main` auto-deploys to Pages (`.github/workflows/pages.yml`). When
shipping a change:

1. Bump `version` in **`package.json`** (the single source of truth —
   `scripts/lib/catalog.mjs` reads it via the exported `VERSION`).
2. Add a `CHANGELOG.md` entry.
3. `npm run lint && npm test && npm run build` — commit the regenerated files
   (CI enforces lint, tests, and that generated files are up to date).
4. Push to `main`; Pages rebuilds.

Note the version model: the deploy stamp shows `major.minor.<git-commit-count>`
(e.g. site reads `0.20.46`), while the hand-set `version` is the semantic
release. So the visible patch number is the commit count, not the hand-set one —
don't be surprised they differ. After deploy, poll
`https://jeancamposlabs.github.io/style-catalog/assets/build.json` to confirm the
`build` number matches `git rev-list --count HEAD`.

## Conventions & gotchas

- **Never hand-edit generated files** (`catalog.json`, `assets/catalog.js`,
  `api/`, `build.*`). Edit the source and rebuild.
- **`assets/app.js` is ES5-flavoured vanilla JS** — no modules, no arrow-heavy
  rewrites, no dependencies. Keep new code in that idiom.
- Effects must be **fully self-contained** (no CDN/external assets), carry a
  `prefers-reduced-motion` guard for motion, and read well in a 16:10 iframe.
- `playwright` in `devDependencies` is only for `npm run posters`; don't pull it
  into runtime code or CI.
- The full vision/roadmap lives in [`PLAN.md`](PLAN.md); the authoring guide in
  [`CONTRIBUTING.md`](CONTRIBUTING.md).
